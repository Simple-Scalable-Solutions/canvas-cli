## Architecture

Generated Go CLI project following printing-press conventions:

- `cmd/canvas-cli/` — CLI entry point (Cobra)
- `cmd/canvas-mcp/` — MCP server entry point
- `internal/cli/` — one file per Canvas resource + five hand-crafted transcendence commands
- `internal/client/` — HTTP client wrapping the Canvas API
- `internal/config/` — TOML config + env-var auth (`CANVAS_LMS_TOKEN`)
- `internal/mcp/` — runtime Cobra-tree MCP surface mirror
- `internal/store/` — SQLite cache with FTS5 full-text search

Config file: `~/.config/canvas-cli/config.toml`  
SQLite DB: `~/.local/share/canvas-cli/data.db`

## Auth

Set `CANVAS_LMS_TOKEN=<api-token>` or run `canvas-cli auth set-token`. The token is sent as `Authorization: Bearer <token>` on every request. Base URL defaults to `https://canvas.instructure.com/api/v1` and can be overridden with `CANVAS_BASE_URL` for self-hosted Canvas instances.

Generated config fields (verified):
- `cfg.CanvasLmsToken` — the primary token field
- `cfg.BaseURL` — default `https://canvas.instructure.com/api/v1`
- `config.Load(configPath string)` — pass `""` for default path

## Key Printing Press Conventions

- Auto-JSON output when piped (no `--json` flag needed)
- `--compact` returns only high-gravity fields (id, name, status, timestamps)
- `--agent` sets all agent-friendly defaults (`--json --compact --no-input --no-color --yes`)
- Typed exit codes: `0`=success, `2`=usage, `3`=not found, `4`=auth, `5`=API error, `7`=rate limited
- `--data-source auto|live|local` controls API vs cached reads

## Transcendence Commands

Five hand-crafted compound commands in `internal/cli/`, registered via `internal/cli/transcendence_register.go`:

| Command | File | Description |
|---------|------|-------------|
| `student-pulse` | `student_pulse.go` | Grade distribution + at-risk students for a course |
| `course-audit` | `course_audit.go` | Quality audit: missing due dates, rubrics, empty pages |
| `grade-export` | `grade_export.go` | Export grades as CSV (student × assignment) |
| `bulk-enroll` | `bulk_enroll.go` | Enroll users from stdin/file into a course |
| `activity-report` | `activity_report.go` | Page views + participations per student |

All transcendence commands share `internal/cli/canvas_client.go` — a lightweight HTTP client with Link-header pagination. To add a new transcendence command:
1. Create `internal/cli/<name>.go` with `newXxxCmd(flags *rootFlags) *cobra.Command` and `runXxx(cc *canvasClient, ..., w io.Writer) error`
2. Add `root.AddCommand(newXxxCmd(flags))` to `registerTranscendenceCommands` in `transcendence_register.go`

## Regenerating the CLI

```shell
# Acquire fresh Canvas OpenAPI spec (community-generated; official spec requires full Canvas build)
# Use: https://github.com/MyzTyn/canvas_lms_rs_openapi_client/raw/main/canvas_lms_openapi.yaml

# Regenerate (WARNING: --force with --output . deletes existing files including .git)
# Use a separate output directory, then merge changes manually
~/go/bin/printing-press generate \
  --spec ./canvas.openapi.yaml \
  --name canvas \
  --owner "Simple Scalable Solutions" \
  --output /tmp/canvas-regen \
  --lenient
```

**After regeneration:** Some generated files may have invalid Go syntax for enum array parameters (e.g., `filter_mode["and"|"or"]`). Fix by converting double-quoted map keys to backtick raw string literals.
